Jadepool

Order

Order is the basic unit for Jadepool to process and record transactions. It is the fundamental of all services in Jadepool.

Order Type

Type Definition
DEPOSIT Transfers made by user to the deposit address given by Jadepool.
WITHDRAW Requested by client to send certain amount of asset to a withdrawal target address.
SWEEP Automatically triggered when balance is higher than the “high water level”configuration.
RECHARGE Transfers made from the cold wallet to the main hot wallet address.
RECHARGE_UNEXPECTED Transfers made directly to the main hot wallet address from other than the cold wallet or transfers made to deposit address of wrong type.
AIRDROP Transfers made from the configured airdrop addresses.
SWEEP_INTERNAL Transfers made towards the internal addresses.
RECHARGE_INTERNAL The order type that corresponds to each “SWEEP INTERNAL” order.
RECHARGE_SPECIAL Currently only used by NEO for claiming gas.
SYSTEM_CALL Orders for certain behaviors. Details are listed below.
PRINCIPAL_FUND Orders for receiving delegation principal.
INTEREST_FUND Orders for receiving delegation reward.
DELEGATE Orders for requesting delegation.
UNDELEGATE Orders for requesting undelegation.


SYSTEM_CALL Definition
UTXO_SCATTER Transfer from hot main address to deposit addresses for creating more utxo.
APPROVE Deposit address authorizes hot main address for using its tokens.
REDELEGATE Switch vote/delegation to another bp/validator.
CLAIM_REWARD Request claiming delegation reward.
SET_REWARD_ADDRESS Set address for receiving delegation reward.


Order State

See Incoming Transfer State Machine and Outcoming Transfer State Machine。

State Definition
init Jadepool receives the withdrawal request but has not yet started processing. Outgoing transfers only.
holding State before sending raw tx to node, orders staying at 'holding' state will be considered unusual and require manual processing. Outgoing transfers only.
online Raw transaction sent to node successfully. Outgoing transfers only.
pending Transaction mined in block and need more blocks produced for comfirmation.
done Final state. Transaction has enough confirmations and is irreversible.
failed Final state. Transaction failed to be mined.


Incoming & Outgoing Transfer

Incoming：order types for transfers that increase Jadepool balance, such as: DEPOSIT, RECHARGE, RECHARGE_UNEXPECTED, RECHARGE_INTERNAL, AIRDROP, RECHARGE_SPECIAL, PRINCIPAL_FUND, INTEREST_FUND, UNDELEGATE
Outgoing：order types for transfers that decrease Jadepool balance, such as: WITHDRAW, SWEEP, SWEEP_INTERNAL, DELEGATE

High Water Level/Target Water Level/Low Water Level

High Water Level：The maximum amount to be stored in the Jadepool hot wallet. If the balance exceeds this level, it will collocate with the target water level to control the amount to be transferred to the cold wallet.
Target Water Level：When the balance in the Jadepool hot wallet exceeds the high water level, certain percentage of the balance will be transferred to cold wallet, and the remaining amount in Jadepool will be at the "target water level". For example, if the high water level is 0.4BTC and the target water level is 0.2BTC, and there are 0.5BTC in the current Jadepool hot wallet, then 0.3BTC should be transferred to the cold wallet, and 0.2BTC will be left in the Jadepool hot wallet.
Low Water Level：The lower limit of the amount in Jadepool hot wallet. If the balance is lower than this level, a warning message will be sent.

The main hot wallet address

The main hot wallet address is not a deposit address and will not be provided to users for deposit (except EOS-like chains and tokens). This address is generally used for gathering and storing changes.

Gathering

Gathering means transfers made from users’ deposit addresses to the main hot wallet address. Taking ETH as an example, each ETH address has a nonce (an incremental serial number for each transaction sent out) and there can only be one input in a transaction. In order to facilitate management of addresses and transactions, Jadepool set the main hot wallet address as the only source to send out transactions (withdrawals and transfer from hot wallet to cold wallet). Thus, only the balance of the main hot wallet is the usable balance, the balance in deposit addresses will be usable only after being gathered to the main hot wallet address.

Total Balance/Available Balance/Unavailable Balance/Frozen Balance/Unfreezing Balance

Total Balance: The sum of the available balance and the unavailable balance.
Available Balance: The amount of token that can be used for outgoing transfers.
Unavailable Balance: The amount of token that cannot be used for outgoing transfers. It is the sum of the tokens scattered at all deposit addresses under "address mode". The unavailable balance is always 0 when under "memo mode".
Frozen Amount: The amount of tokens that have been delegated/staked. It is not included in the total balance.
Unfreezing Amount: The amount of tokens being undelegated/unstaked. Each chain has different period for undelegation. It could take three days at least or as long as twenty-one days. During this time, tokens are in the state of unfreezing. The unfreezing amount is not included in the total balance.

Callback

The callback is the way Jadepool notifies client of the order status and audit results. For each service (appid), client can set notification URL for each order type on Admin.

Audit

Jadepool auditting is to find the nearest block in database based on the auditting timestamp given by client by calling the audit API. Then find all orders complete since the last auditting timestamp.

ECC

ECC is an encryption algorithm used by Jadepool for securing communication between internal services and with client. The service to send data should have a private key to sign, and the service to receive data needs to have the pair public key of the message sender to verify the signature. Take the communication between Jadepool and the client as an example, both services need to send and receive data. Jadepool and the client should exchange their own public key. This can be done on admin.
